title:mysql5.7 json类型
date:2017-02-10 06:35:49
mysql5.7 新增了json类型，观其思路旨在代替memcache来做对象存储
– insert into test (data) values (“[1,2,3,4]”) – 插入
– select from test where data->’$[0]’=1; – $[0]访问第一个元素
– insert into test (data) values (‘{“k1”: “value”, “k2”: 10}’)
– select from test where data->”$.k2” >9 – $.k2 访问名为k2的元素
– select 1+1;
– select json_object(“aaa”,1) – 组合json对象
– select json_object(1,2,3,4)
– set @j = json_object(“aaa”,1);
– select @j;
– select json_array(1,2,3,4,5) – 组合json数组
– select json_valid(“[1,2,3,4]”) – 验证是否为json
– select json_merge(“1”,”2”)
– select json_extract(“[1,2,3]”,”$[0]”) – 获取json信息
– select json_extract(‘{“a”:3}’,”$.a”)

title:pptpd限制单用户登陆脚本
date:2016-03-15 04:15:01

将下面代码存为auth-up，放到/etc/ppp/目录下并加入可执行权限。

#!/bin/sh
# get the username/ppp line number from the parameters
REALDEVICE=$1
USER=$2
# create the directory to keep pid files per user
mkdir -p /var/run/pptpd-users
# if there is a session already for this user, terminate the old one
if [ -f /var/run/pptpd-users/$USER ]; then
kill -HUP `cat /var/run/pptpd-users/$USER`
fi
# copy the pid file of current user to /var/run/pptpd-users
cp "/var/run/$REALDEVICE.pid" /var/run/pptpd-users/$USER

title:ssh端口转发
date:2016-08-28 07:04:54
本机为B ，A、C两台远程机器，

1.将A的80端口映射到C的8080端口上：

sudo ssh -R 8080:A的IP:80 root@B的IP（B可以换成A/C）

2.将A的80端口映射到本地的8080端口：

sudo ssh -L 80:127.0.0.1:8080 root@A的IP
